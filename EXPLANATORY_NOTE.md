# Разработка веб\-приложения «Wishlist App» (Система управления списками желаний)

# Введение и постановка задачи

Целью данного проекта является разработка многопользовательского веб\-приложения для создания, хранения и совместного использования списков желаний (вишлистов). Проект направлен на демонстрацию практических навыков работы с современными веб\-технологиями, паттернами проектирования и архитектурными стандартами.

# Основные технические требования к реализации:

* Язык программирования: PHP 8.3.  
* Фреймворк: Laravel 11.x.  
* База данных: SQLite (взаимодействие через Eloquent ORM и нативный PDO).  
* Архитектура: Модульный монолит с соблюдением стандартов PSR-1, PSR-12 и автозагрузкой классов по стандарту PSR-4.  
* Безопасность: Защита от SQL-инъекций (Prepared Statements), CSRF-атак, XSS-уязвимостей. Хэширование паролей.  
* Ролевая модель: Разграничение прав доступа (RBAC) на основе ролей (пользователь, редактор, администратор).  
* Обработка исключений: Изолированный перехват HTTP-ошибок с динамической генерацией заголовков и статус-кодов.  
1. Архитектура приложения и изолированные модули  
   Приложение построено на базе архитектурного шаблона MVC (Model-View-Controller).

Для демонстрации навыков проектирования сложных систем, функционал технической поддержки (сбор и обработка отчетов об ошибках) вынесен в независимый локальный пакет local/support-module.  
Данный пакет имеет собственный конфигурационный файл composer.json и интегрирован в основное ядро через механизм Dependency Injection (Service Locator). Для работы с базой данных внутри пакета используется исключительно расширение PDO (PHP Data Objects), что гарантирует низкоуровневый контроль над SQL-запросами.

# Ролевая модель и права доступа (RBAC)

В системе реализовано строгое разграничение прав доступа с использованием перечислений (Enum) и механизма политик (Policies/Gates):

* Guest (Гость): Доступ только к публичным разделам (авторизация, регистрация). Доступ к отправке анонимных отчетов об ошибках.  
* User (Пользователь): Создание, редактирование и удаление собственных желаний. Добавление других пользователей в друзья. Бронирование публичных желаний друзей (предотвращение дублирования подарков). Просмотр ответов технической поддержки в личном кабинете.  
* Editor (Редактор): Права обычного пользователя, а также возможность удаления любого публичного контента в целях модерации.  
* Admin (Администратор): Полный доступ к системе. Просмотр списка всех зарегистрированных пользователей, изменение их ролей. Доступ к панели обработки системных ошибок и жалоб от пользователей с возможностью обратной связи.

# Работа с формами и файловой системой

В проекте реализована безопасная загрузка файлов. При добавлении желаний или отправке отчетов об ошибках, пользователь может прикрепить графический файл или PDF-документ.  
На стороне сервера проводится валидация типа файла (mimes: jpg, jpeg, png, pdf) и его размера (до 5 МБ для отчетов). Файлы сохраняются в изолированном публичном диске (storage/app/public) с использованием фасада Storage, что защищает систему от выполнения вредоносного кода. Доступ к файлам осуществляется через безопасные символические ссылки.

# Конфигурация и внедрение зависимостей

Конфигурация проекта (подключение к БД, настройки сессий и почты) вынесена в переменные окружения (.env). Для обеспечения модульности применяется шаблон Dependency Injection. Внутри сервис-провайдера (AppServiceProvider) происходит инициализация объекта PDO с передачей конфигурационных данных базы, после чего экземпляр класса ReportManager регистрируется в DI-контейнере Laravel как Singleton. Это позволяет контроллерам получать готовый к работе сервис без жесткой привязки к его внутренней реализации.
