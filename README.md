# Wishlist App

Многопользовательское веб-приложение для управления списками желаний с функциями добавления в друзья, бронирования подарков и модулем технической поддержки (с поддержкой RBAC).

## Выполненные архитектурные требования
- **Модульность:** Функционал технической поддержки вынесен в кастомный локальный пакет (`packages/support-module`) с отдельным `composer.json` и PSR-4 автозагрузкой.
- **База данных:** Работа через чистый `PDO` в модуле поддержки + Eloquent ORM в основном приложении.
- **DI / Service Locator:** `ReportManager` инициализируется конфигурацией БД и внедряется через контейнер зависимостей (AppServiceProvider).
- **Обработка ошибок:** Кастомный рендер исключений (404, 403, 500) с возвратом HTTP-кодов, отдачей JSON для API и внедрением заголовков (`X-Error-Time`, `X-Error-Code`). Интеграция с формой обратной связи на страницах ошибок.
- **Файловая система:** Строгая валидация и загрузка файлов (mimes: jpg, png, pdf, max: 5MB) с использованием фасада Storage.
- **Тестирование:** Написаны Feature-тесты для проверки модуля кастомных ошибок, заголовков и роутов.

## Запуск проекта (Docker / Laravel Sail)

Для запуска приложения требуется только установленный **Docker** и **Docker Compose**. Локальная установка PHP не обязательна.

**1. Клонирование репозитория и переход в директорию:**
```bash
git clone https://github.com/ingvionio/Golybev-Kostyuk-PHP
cd wishlist-app
cp .env.example .env
```

**2. Установка зависимостей (через временный Docker-контейнер):**
Если на ПК не установлен PHP/Composer, выполните команду для установки зависимостей:
```bash
docker run --rm \
    -u "$(id -u):$(id -g)" \
    -v "$(pwd):/var/www/html" \
    -w /var/www/html \
    laravelsail/php83-composer:latest \
    composer install --ignore-platform-reqs
```
*(Если PHP установлен локально, достаточно выполнить `composer install`)*.

**3. Запуск контейнеров в фоновом режиме:**
```bash
./vendor/bin/sail up -d
```

**4. Установка NPM-зависимостей и сборка фронтенда:**
```bash
./vendor/bin/sail npm install
./vendor/bin/sail npm run build
```

**5. Настройка базы данных и хранилища файлов:**
По умолчанию проект использует SQLite. Создадим файл БД, сгенерируем ключ, выполним миграции с сидерами и создадим симлинк для картинок:
```bash
touch database/database.sqlite
./vendor/bin/sail artisan key:generate
./vendor/bin/sail artisan migrate:fresh --seed
./vendor/bin/sail artisan storage:link
```

Проект доступен по адресу: **http://localhost**

## Тестовые доступы

При выполнении миграций с ключом `--seed` в базе создаются аккаунты для проверки системы распределения ролей (RBAC):

**Администратор:**
- Email: `admin@example.com`
- Пароль: `password`
*(Доступ к `/admin`, управление ролями пользователей, обработка входящих жалоб с прикрепленными файлами, модерация контента).*

**Пользователь:**
- Email: `user@example.com`
- Пароль: `password`
*(Создание желаний, добавление друзей, бронирование, отправка отчетов об ошибках со скриншотами).*

Для проверки кастомной страницы ошибок и тестирования формы отправки отчета, перейдите по любому несуществующему маршруту (например, `http://localhost/test-404-page`).
